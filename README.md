# Linux Camera-based Adaptive Brightness (LCAB)

这是一个专为 Linux 笔记本设计的极致轻量级自动亮度控制方案。它通过笔记本摄像头捕获环境光，并应用视觉校正算法，实现类似手机的平滑自动亮度调节体验。

## 🌟 核心特性

- **变频自适应采样**：
  - **基础模式 (1Hz)**：环境稳定时每秒采样一次，指示灯瞬间闪烁，极致省电。
  - **爆发模式 (10Hz)**：检测到目标亮度与当前亮度差距超过 20% 时，自动触发高频采样，确保极速响应环境变化。
- **中央重点加权测光**：
  - 默认对画面中心及中下部（人脸常驻区域）分配 **80% 的权重**，有效排除背景强光（如身后的窗户）对人眼感官的干扰。
- **人眼视觉校正 (Gamma Correction)**：
  - 应用 **1.8 Gamma 曲线** 映射，使屏幕亮度的增减完全符合 Weber-Fechner 人眼感官定律，过渡自然不突兀。
- **极致性能与功耗优化**：
  - **生产级守护进程 (C Daemon)**：纯整型运算，预计算查找表 (LUT)，零浮点开销，零标准输出 IO。
  - **隐私友好**：采用离散采样技术，摄像头仅在采样瞬间（约 100ms）开启，物理指示灯平时熄灭。
- **无缝平滑过渡**：
  - 实时读取系统物理亮度作为平滑滤波基准，确保无论程序重启还是模式切换，亮度变化始终如一。

## 📂 项目结构

| 文件名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `autobright_daemon.c` | **生产版** | 极致精简、无输出、整型优化版，适合作为后台服务长期运行。 |
| `solution.c` | **调试版** | 包含详尽中文日志输出与浮点运算，适合在终端运行观察测光逻辑。 |
| `camera_auto_brightness.py` | **实验版** | 基于 Python + FFplay，支持实时可视化区域标记预览，方便参数调优。 |
| `autobright_fixed_point.c` | **验证版** | 带有日志输出的整型算法版本，用于验证定点数运算的精度。 |
| `Makefile` | **构建脚本** | 支持一键编译调试版或守护进程版。 |

## 🛠️ 前置要求

确保您的 Linux 系统已安装以下工具：
- `v4l-utils` (用于控制摄像头参数)
- `brightnessctl` (用于控制屏幕背光)
- `gcc` & `make` (用于编译 C 程序)
- `ffmpeg` & `python3` (仅运行 Python 实验版需要)

## 🚀 快速开始

### 1. 编译所有版本
```bash
make
```

### 2. 调试运行 (观察实时测光与逻辑)
运行带日志输出的调试版：
```bash
./autobright
```

### 3. 生产部署 (静默运行)
直接启动极致优化的守护进程版：
```bash
./autobright_daemon
```

### 4. 视觉化参数调优 (Python)
如果您想通过预览窗口微调测光区域：
```bash
python3 camera_auto_brightness.py -v --fps 15
```

## ⚙️ 参数微调 (宏定义)

在 `autobright_daemon.c` 顶部，您可以根据硬件差异微调以下宏：

- `EXP`: 固定曝光值 (默认 2000)。
- `GAIN`: 固定增益 (默认 100)。
- `WEIGHT`: 关注区权重百分比 (默认 80%)。
- `GAMMA`: 视觉校正系数 (建议 1.5 - 2.2)。
- `BR_THRESHOLD`: 触发爆发模式的亮度差值阈值 (默认 20%)。
